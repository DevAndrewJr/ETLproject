## Abilitare le connessioni remote a SQL Server
Per consentire le connessioni remote al server SQL Server, seguire i seguenti passaggi:
#### 1. Accesso alle proprietà del nodo/istanza:
Utilizzare SQL Server Management Studio (SSMS) per connettersi al server SQL Server.
Fare clic con il pulsante destro del mouse sull'istanza desiderata e selezionare "Proprietà".
#### 2. Attivazione delle connessioni remote:
Nella finestra di dialogo delle proprietà, passare alla sezione "Connessioni".
Assicurarsi che la casella di controllo "Allow remote connections to this server" sia selezionata. Se non è già 
selezionata, attivarla.
#### 3. Applicazione delle modifiche:
Fare clic su "OK" per confermare le modifiche e chiudere la finestra delle proprietà.
#### 4. Riavvio del servizio SQL Server (opzionale):
Se necessario, riavviare il servizio SQL Server per applicare completamente le modifiche.

## Configurazione della strategia di denominazione dei campi delle tabelle
Per impostazione predefinita, Hibernate in Spring Boot utilizza una strategia di denominazione dei campi delle tabelle 
chiamata "snake_case". Se i nomi delle tabelle nel database utilizzano una convenzione diversa, come ad esempio il 
"camelCase", è necessario configurare Hibernate per utilizzare la strategia corretta.
Per farlo, è possibile specificare la strategia desiderata nel file di configurazione di Spring Boot (solitamente 
application.properties o application.yml), utilizzando una delle seguenti proprietà:

properties
spring.jpa.hibernate.naming-strategy=org.hibernate.cfg.ImprovedNamingStrategy
oppure, se si utilizza Spring Boot 2.x o versioni successive:

properties
spring.jpa.hibernate.naming.physical-strategy=org.hibernate.boot.model.naming.PhysicalNamingStrategyStandardImpl
Assicurarsi di includere le dipendenze Hibernate corrette nel progetto e verificare che la configurazione abbia avuto 
l'effetto desiderato controllando i nomi delle colonne nel database dopo il riavvio dell'applicazione.

Con queste configurazioni, il progetto Spring dovrebbe essere pronto per connessioni remote a SQL Server e per una 
corretta mappatura dei nomi dei campi delle tabelle.
